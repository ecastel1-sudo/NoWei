# Citations and References for MoE Competition Submissions

This file lists concise, authoritative references for the algorithmic components
used across the repository and the `nowei` submission variants. Use these when
preparing writeups or attribution statements.

## 1) DeepSeek / EPLB (reference implementation)
- DeepSeek EPLB implementation (code): https://github.com/deepseek-ai/EPLB/blob/main/eplb.py
- Purpose: replication + balanced packing + hierarchical placement for expert parallelism.

## 2) submission_v2_dual.py — DUAL‑EWMA DRIFT (STARE‑LB v2)
- Core ideas:
  - Dual EWMA (fast + slow) for drift detection and selective rebalancing.
  - Placement generated by balanced packing/replication; optional anchoring to reduce churn.
- Background / references:
  - EWMA control charts (classic practical reference): https://en.wikipedia.org/wiki/Exponentially_weighted_moving_average
  - Drift detection / adaptive windows (ADWIN): Bifet, A. & Gavaldà, R., "Learning from time‑changing data with adaptive windowing" (SDM 2007). https://www.sciencedirect.com/science/article/pii/S0167923607000073
  - Assignment anchoring (greedy relabeling idea) — related to the assignment problem (Hungarian method): Kuhn, H.W., "The Hungarian Method for the assignment problem", 1955. https://en.wikipedia.org/wiki/Hungarian_algorithm

## 3) submission_v3_reward.py — REWARD GATE (STARE‑LB v3)
- Core ideas:
  - Greedy cost‑benefit gate using the competition cost model: seconds_saved (ΔPAR * compute_seconds) vs seconds_spent (moves * bytes / BW * SAFETY).
  - Accept layer moves when net_seconds > 0; sort by net benefit.
- Background / references:
  - Greedy algorithms / knapsack intuition: Cormen et al., "Introduction to Algorithms" (greedy strategies). https://mitpress.mit.edu/books/introduction-algorithms-third-edition
  - Practical cost‑benefit / reward gating is a standard myopic optimization approach in online resource allocation; see scheduling & greedy heuristics surveys.

## 4) submission_v4_anchored.py — REWARD + ANCHORED (STARE‑LB v4)
- Core ideas:
  - Same reward gate as v3, plus an anchored packing step that permutes device labels to maximise overlap with the current layout, reducing physical moves without altering PAR.
- Background / references:
  - Jaccard / overlap heuristics for relabeling: Jaccard index overview — https://en.wikipedia.org/wiki/Jaccard_index
  - Exact assignment viewpoint: Hungarian method (Kuhn, 1955) for optimal relabeling.

## 5) Common algorithmic building blocks (used in DeepSeek + submissions)
- Replication to reduce peak load and balanced packing to distribute replicas across devices.
  - "Power of Two Choices" / balanced allocation literature (benefits of multiple-choice allocation): Azar, Broder, Karlin, Upfal (1999) and Mitzenmacher surveys. Example survey: https://www.eecs.harvard.edu/~michaelm/postscripts/my-survey.pdf
- Bin‑packing / multiway balanced partitioning references: standard algorithm / approximation literature (e.g., Korf, Coffman & Garey texts).

## 6) Transmission / cost model and PAR metric
- In‑repo model: transmission_time = moves * EXPERT_BYTES / BANDWIDTH; modeled_total_time = BALANCED_COMPUTE_SECONDS * mean_PAR + transmission_time.
- Related literature: workload makespan and load imbalance metrics — Graham, R.L., "Bounds for Certain Multiprocessing Anomalies" (1966) and surveys on scheduling and load balancing.

## 7) Mixture‑of‑Experts (context)
- Foundational MoE papers for context (not directly implementing placement):
  - Shazeer, N., et al., "Outrageously Large Neural Networks: The Sparsely‑Gated Mixture‑of‑Experts Layer" (2017). https://arxiv.org/abs/1701.06538
  - Fedus, W., et al., "Switch Transformers: Scaling to Trillion Parameter Models with Simple and Efficient Sparsity" (2021). https://arxiv.org/abs/2101.03961
